## 三、前 3 周：活下来，搞明白
**阶段目标**：把"该找谁、要交什么、用什么平台/算力/数据"弄明白，别慌。先搞清这条线是研究导向还是业务导向。

关键行动：
- [ ] 找 leader 对齐前 3 月最看重的 3 件事（研究导向？业务导向？），写下来
- [ ] 画出人际地图（leader/带教/数据 owner/平台/产品接口人），记到备忘录
- [ ] 跑通实验平台 + 申请到算力 + 拿到数据权限 + 摸清 Eval 流程
- [ ] 读一篇组内核心论文，用自己话复述核心思想
- [ ] 搭好 AI 研发基础设施：提示词库 + 实验记录模板 + 论文笔记库

**AI 时代提示**：这阶段就用 AI 把"解释论文、整理术语表、写 baseline 草稿"接过去，给自己省出"搞明白"的时间。但别让 AI 替你想"该问 leader 什么、该选哪个 baseline"——这些得你自己定。

**检查点（第 3 周末）**：我能说清 leader 的核心期待（研究还是业务导向）吗？遇到实验/数据/算力问题知道找谁吗？我读过一篇核心论文或跑通过一个 baseline 吗？
**危险信号**：三周还不敢问"参数为啥这么设" / 忙但说不清忙啥 / 完全没碰过实验平台或没申请到算力。

## 四、前 3 月：立住，出成绩

### 第 1 月：独立跑通一个实验 / 交付一个功能
关键行动（按岗位族选）:
- [ ] 算法/研究：独立 repro 一个 baseline，把指标跑出来（哪怕没涨点）
- [ ] 数据/ML 工程：独立跑通一条训练/推理管线，交付一个可用模块
- [ ] AI 产品/运营/售前：交付一个能 demo 的功能 / 一次增长实验 / 一份客户场景库
- [ ] 主动同步进度，让 leader 知道你卡在算力/数据/设计

### 第 2 月：对齐"效果 vs 上线"的取舍
关键行动：
- [ ] 接活先确认"做成什么样算好"（抠指标还是保上线）
- [ ] 研究岗做 ablation 理解涨因为啥；工程岗把脚本改成自己设计的流程
- [ ] 产品/售前：把"技术能做什么"翻译成"用户/客户要什么"
- [ ] 做一件"超出预期一点点"的事（补实验记录模板 / 写 onboarding 文档）
- [ ] 验证 AI 输出，记一次翻车案例进 repro 库

### 第 3 月：扛事 + 想方向
关键行动：
- [ ] 认领一个稍大的任务或改进
- [ ] 写一份自己的前 3 月方法论备忘（实验规范 / 管线 checklist）
- [ ] 认真想一次：做研究 / 做业务 / 转产品·售前？

**AI 时代提示**：第 3 月务必做一次"脱离 AI 裸做"——关掉 AI 手写一个推导或裸跑一个脚本，确认地基还在。否则你只是个"很能交付的脆皮"。

**检查点（第 3 月末）**：leader 愿意交重要不紧急的模块吗？有带我印记的成果吗？脱离 AI 还能干吗？对做研究/业务/转方向有判断了吗？
**危险信号**：三个月只跑别人脚本从没自己设计实验 / 评价停在"态度好" / 从没读核心论文讲不清原理 / 从没想过方向。
