# Landing 清单（前 3 周 / 前 3 月 · AI 行业版）

> 每天/每周勾一点。清单不是为了凑数，是为了让你知道"我今天有没有在推进落地"。
> 适用：算法/研究、数据/ML 工程、AI 产品、AI 运营/增长、解决方案/售前。按你的岗位族勾对应的项。

## 前 3 周
- [ ] 找 leader 对齐前 3 月最看重的 3 件事（研究导向？业务导向？）
- [ ] 画出人际地图（leader/带教/数据 owner/平台/产品接口人）
- [ ] 跑通实验平台 + 申请到算力 + 拿到数据权限
- [ ] 摸清 Eval / 评测流程
- [ ] 读一篇组内核心论文，用自己的话复述核心思想
- [ ] 搭好 AI 研发基础设施：提示词库 + 实验记录模板 + 论文笔记库
- [ ] 本周问出一个"傻问题"（如"这参数为啥这么设"）并记录没人笑你

## 前 3 月 · 第 1 月
- [ ] 算法/研究：独立 repro 一个 baseline，指标跑出来
- [ ] 数据/ML 工程：独立跑通一条训练/推理管线，交付模块
- [ ] AI 产品/运营/售前：交付一个能 demo 的功能 / 一次增长实验 / 客户场景库
- [ ] 交出的东西自己先检查（指标对不对、脚本能不能复现）
- [ ] 发出第一份主动进度同步（让 leader 知道你卡在哪）

## 前 3 月 · 第 2 月
- [ ] 接活前先确认"好"的标准（抠指标还是保上线）
- [ ] 研究岗做一次 ablation；工程岗把脚本改成自己设计的流程
- [ ] 产品/售前：做一次"技术→人话"的转述练习
- [ ] 做一件超出预期的小事（补实验记录模板 / 写 onboarding 文档）
- [ ] 验证一次 AI 输出，记进 repro / 翻车记录

## 前 3 月 · 第 3 月
- [ ] 认领一个稍大的任务或模块
- [ ] 写一份自己的方法论备忘（实验规范 / 管线 checklist）
- [ ] 做一次"脱离 AI 裸做"自检（手写推导 / 裸跑脚本）
- [ ] 认真想一次方向：做研究 / 做业务 / 转产品·售前？

## 每周复盘
- 这周做了：__
- 卡在哪（算力 / 数据 / 设计 / 人际）：__
- 下周重点：__
- 一个 AI 帮到我的点 / 一个 AI 坑了我的点：__
