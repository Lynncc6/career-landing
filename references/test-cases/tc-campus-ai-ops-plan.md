# 我的 Landing 计划（前 3 周 / 前 3 月 · AI Agent 产品运营版）

> 测试案例 persona：小柒，校招应届生，刚入职一家做 AI Agent 的公司做产品运营（类似 WorkBuddy 那种），入职第 1 周。  
> 痛点：人际融入（不敢问傻问题）/ 任务预期不清 / 技术迭代太快追不动 / 方向迷茫 / 不会用 AI 提效。有 mentor、团队鼓励提问。  
> 调性：人话 + 有观点。

## 一、我是谁 / 我在哪（诊断结论）

- 身份：应届生（校招，几乎没职场经验，但实习摸过一点运营）
- 环境：AI Agent 公司（产品运营，类似 WorkBuddy）· AI 运营/增长岗
- 入职阶段：第 1 周（刚入职）
- 最卡的 3 件事：人际融入（不敢问傻问题）/ 任务预期不清（不知道领导要我干啥）/ 技术迭代太快（AI 产品更新快，追不动也怕露怯）
- 资源：mentor=有；团队氛围=鼓励提问；用 AI 习惯=用 AI 写文档改文案，但没深度用在运营分析上

## 二、痛点大实话（写给自己）

- "不敢问傻问题" → 大实话：我怕一问就暴露"我是小白"。但第 1 周没人指望你懂，问不出才真露怯。
- "追不动 AI 产品迭代" → 大实话：功能一周一个样，我怕跟不上显得没用。但运营要的不是比算法懂技术，是比谁都懂"用户拿这产品到底干啥、卡在哪"——这是他们追不过我的地方。
- "不知道运营在这值钱啥" → 大实话：校招容易把自己当打杂。产品运营的命根是"把用户声音翻译成产品能用的输入"，这活算法和产品都缺，且不会被 AI 自动干掉。

## 三、前 3 周：活下来，搞明白

**阶段目标**：别慌，先把"该找谁、要交什么、我们的 agent 到底能干嘛"弄明白。前两周不出成绩没人怪你，但"听不懂、问不出、讲不清产品边界"会让人觉得你不行。

关键行动：

- [ ] 找 leader 对齐前 3 月最看重的 3 件事——产品运营是业务导向（拉新/活跃/留存/用户反馈闭环），写下来
- [ ] 画人际地图：leader / 带教 / 算法接口人 / 产品 / 研发，先知道"用户反馈卡了找谁、数据找谁"
- [ ] 摸环境：产品后台、数据看板、用户反馈渠道、A/B 实验入口、agent 能力边界文档，跑通一遍
- [ ] 把你们 agent 的核心能力和边界读一遍，能用大白话讲清"我们的 agent 能干什么、不能干什么"（产品运营的命根，必须自己懂）
- [ ] 搭好 AI 运营基础设施：提示词库（用户分群/写文案/分析）+ 用户反馈整理模板 + 运营周报模板
- [ ] 用 AI 把"用户反馈分类、写周报草稿"接过去，省出时间搞明白

**AI 时代提示**：这阶段让 AI 帮你把用户反馈自动分类、写运营周报草稿。但"我们的 agent 到底能干嘛、用户到底要什么"得你自己懂——别让 AI 替你想"该问 leader 什么"。

**检查点（第 3 周末）**：我能说清 leader 对前 3 月的核心期待吗？遇到反馈/数据问题知道找谁吗？我能不靠文档大白话讲清产品能力边界吗？  
**危险信号**：三周还不敢问傻问题 / 忙但说不清忙啥 / 完全没碰过产品后台和数据看板 / 还不懂自己产品能干嘛。

## 四、前 3 月：立住，出成绩

### 第 1 月：独立交付小运营动作，立"靠谱"人设

关键行动：

- [ ] 算法/产品运营：独立发一次用户调研 / 梳理一份反馈分类 / 跑一个小 A/B 实验
- [ ] 交出的东西自己先检查（数据对不对、结论站不站得住）
- [ ] 发出第一份主动进度同步：让 leader 知道你在动、卡在哪
- [ ] 用 AI 放大产出，但"这周用户反馈说明啥"这种判断自己拍板

### 第 2 月：对齐"运营效果怎么衡量" + 用 AI 但不被带偏

关键行动：

- [ ] 接活先确认"做成什么样算好"——是看活跃、留存还是反馈闭环
- [ ] 做一件超出预期的小事：把散落的 user feedback 整理成算法/产品能直接用的结构化输入模板
- [ ] 验证 AI 给的运营结论（它给的"用户想要 X"可能瞎编，交叉看原始反馈）
- [ ] 用 AI 做用户分群/增长分析，但结论留痕

### 第 3 月：扛事 + 想方向

关键行动：

- [ ] 认领一个运营模块（某垂类 / 某渠道 / 某反馈闭环）
- [ ] 写一份自己的前 3 月方法论备忘（用户反馈→产品的输入规范）
- [ ] 认真想一次：深耕产品运营 / 往 AI 产品靠 / 往数据分析靠？

**AI 时代提示**：第 3 月做一次"脱离 AI 裸做"——关掉 AI 自己写一份用户反馈洞察，确认领域判断还在。否则你只是个"很能交周报的脆皮"。

**检查点（第 3 月末）**：leader 愿意交一个运营模块给你吗？有带我印记的成果（一份反馈模板/一次调研）吗？脱离 AI 还能写洞察吗？对"运营值钱在哪"有判断了吗？  
**危险信号**：三个月还不敢问傻问题 / 只读二手资讯不懂自己产品边界 / 从没靠近过真实用户 / 评价停在"态度好"。

## 五、资源与 mentor 行动项

- 本周要主动找谁：带教（约 30 分钟过一遍"前 3 月我最该搞懂的 3 件事"）；算法接口人（问"你们最缺运营侧什么输入"）
- 想从 mentor 那要的一样东西：一份"新人前 3 周该读懂的产品/agent 文档清单"
- 我能给别人的价值：把用户反馈变成结构化输入——这是你最便宜也最值钱的输出

## 六、一句话总结

前 3 周先别装懂，把人、后台、数据、产品边界摸透；前 3 月用 AI 抢时间但把"用户要什么、产品边界在哪"的判断留给自己。算法追技术，我追用户——这是他们抢不过我的位置。
