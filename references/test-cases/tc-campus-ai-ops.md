# 测试案例：校招入职 AI Agent 产品运营（类似 WorkBuddy）

> 用途：作为 `career-landing` skill 的可复用回归/演示用例。覆盖「校招应届生 + AI Agent 运营岗 + 刚入职（第 1 周）」这一最标准场景。
> 关联产物：`tc-campus-ai-ops-plan.md`（计划）、`tc-campus-ai-ops-checklist.md`（清单）。

---

## 1. 测试目标
验证 skill 在以下条件下的诊断与产出质量：
- 受众命中：AI 行业 · AI 运营/增长岗（产品运营）
- 身份命中：应届生（校招，几乎无经验）
- 阶段命中：刚入职（前 3 周窗口，走标准「前 3 周 / 前 3 月」框架，而非第 4 月起的重建框架）
- 痛点命中：人际融入 + 任务预期不清 + 技术迭代太快 + 方向迷茫（校招新人典型组合）
- 资源命中：有 mentor、团队鼓励提问
- AI 习惯命中：用 AI 写文档但未深度用于运营分析

验收标准：
1. 诊断能覆盖 6 维，且痛点识别到「不敢问傻问题 / 追不动迭代 / 不知运营值钱啥」。
2. 计划重心正确加权：岗位族=AI 运营 → 加重「把技术翻成人话（搞清产品能力边界）」「靠近用户与场景」「用 AI 做增长实验」；身份=应届生 → 加重「补 AI 底层认知 + 靠近厉害的人」；痛点含人际融入 → 加重「人际地图 + 主动同步」。
3. 产出是个性化、非套话，调性「人话 + 有观点」。

---

## 2. 触发输入（模拟用户原话）
> "我刚校招进了一家做 AI Agent 的公司做产品运营，类似 WorkBuddy 那种，才入职第 1 周。啥都不懂，怕问傻问题显得小白，也不知道领导到底要我干啥。感觉 AI 产品更新太快了，我追不动也怕露怯。能用这个 skill 帮我规划下吗？"

---

## 3. 诊断脚本（6 维，分批问，每轮 1-3 题）

**第 1 轮（身份 + 环境）**
- Q：你怎么到这个岗的？→ A：校招应届生，实习摸过一点运营。
- Q：公司类型和岗位族？→ A：AI Agent 公司，产品运营（类似 WorkBuddy）。

**第 2 轮（入职阶段 + 资源）**
- Q：现在入职多久？→ A：第 1 周。
- Q：团队有人带吗？氛围呢？→ A：有 mentor，团队鼓励提问。

**第 3 轮（痛点，多选）**
- Q：现在最卡的 3 件事？→ A：不敢问傻问题（人际融入）/ 不知道领导要啥（任务预期不清）/ 追不动 AI 产品迭代（技术迭代太快）；隐含方向迷茫。

**第 4 轮（AI 习惯）**
- Q：平时用 AI 辅助工作到什么程度？→ A：用 AI 写文档改文案，但没深度用在运营分析上。

---

## 4. 期望产出（验收对照）
- 计划文档：标准「前 3 周 / 前 3 月」结构；环境节显式标注 AI Agent 产品运营；前 3 周含「读懂 agent 能力边界、能用大白话讲清产品能干嘛」；前 3 月按运营岗定制（发调研/跑 A-B/反馈结构化）；危险信号含「不懂自己产品边界 / 从没靠近真实用户」。
- 清单：前 3 周含「跑通后台+看板+读懂能力边界」；第 2 月含「把 feedback 整理成结构化输入模板」；第 3 月含「脱离 AI 裸写用户洞察」。
- AI 时代提示：用 AI 做用户分群/分析，但「产品边界 + 用户要什么」判断留给自己。
