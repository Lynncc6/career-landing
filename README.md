# career-landing

> AI 行业新人落地（landing）规划 skill。先诊断你的现状，再给你一份「前 3 周 / 前 3 月」专属计划 + 可勾选清单。

## 为什么需要它

AI 时代有个隐蔽的坑：年轻人以为自己没有 landing 期了。

公司不教了、初级岗被 AI 抽走、产出门槛被拉平——你一进场就被按着头"看起来 80 分"。但"能交付"和"真懂"之间的 gap，从来没人帮你补。三个月后，一堆"很能交付的脆皮专家"。

在 AI 行业这坑更狠：新模型一上你调的 prompt 可能就废了、实验跑不出结果不知道是数据问题还是自己虚、研究 vs 落地天天撕扯。

**landing 期没消失，只是从公司安排好的"安全缓冲"变成了你得自己设计的私人项目。** 这个 skill 就是帮你设计它的。

## 适合谁

刚进入 AI 行业的人，覆盖五类岗位族：

| 岗位族 | 典型场景 |
|---|---|
| 算法 / 研究 | 应届进大模型团队，跑不动实验、数学虚 |
| 数据 / ML 工程 | 只跑别人写的脚本，没设计过完整管线 |
| AI 产品 | 搞不清模型能力边界，需求和技术对不上 |
| AI 运营 / 增长 | 改 prompt、收 bad case，感觉自己像"AI 保洁员" |
| 解决方案 / 售前 | 技术翻不成人话，客户问深了就露怯 |

不管你是校招应届、社招转行、还是从别的职能转岗过来，都能用。

## 框架说明

### 前 3 周：活下来

目标不是表现，是搞清楚游戏规则——领导要什么、该找谁、流程怎么走、AI 怎么帮你省重复劳动。在 AI 行业还要额外搞清：这条线是研究导向还是业务导向、算力和数据权限找谁要、核心论文读没读。

### 前 3 月：立住

| 月份 | 核心动作 |
|---|---|
| 第 1 月 | 独立交付小任务，立"靠谱"人设；主动同步进度 |
| 第 2 月 | 主动对齐预期；做超出预期一点点的事；验证 AI 输出 |
| 第 3 月 | 承担稍大模块；提改进；沉淀方法论；复盘方向 |

### AI 时代加成模块

贯穿全程的核心原则：**用 AI 加速，但别被 AI 架空成脆皮。**

- 让 AI 解释论文、写 baseline，但实验设计与结论自己负责
- 定期"脱离 AI 裸做"——补数学底层，做推导不留辅助
- 用 AI 搭基础设施（提示词库、知识库、常用模板），把时间留给"搞明白"
- 建立"验证习惯"：AI 给的东西，先想怎么验它靠不靠谱

## 输出调性

所有面向用户的内容——诊断对话、计划文档、清单——统一用**「人话 + 有观点」**风格：

- 像直白的资深朋友，敢给判断（如"靠谱比聪明值钱"）
- 不写 HR 手册腔，不堆术语
- 有态度，但保持尊重与可操作
